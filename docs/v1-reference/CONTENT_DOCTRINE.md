# Gangatirtham — Content Doctrine

> How the archive is written, reported, shot, checked, translated, and published. The doctrine is **locked**. The canonical exemplar is the **Varanasi / Kashi** page — every place is written against it.

## The principle

When a visitor lands on a place page, they should feel they have **encountered the river, not read about it.** A page must deliver: **presence, place, humanity, memory, the river.** The goal is never to *explain* a place.

## The seven rules

1. **No two places tell the same story.** Each carries a *distinct truth* (birth, naming, labour, death, the wound, the fracture, the return). Range is the whole game.
2. **Encounter, not explainer.** Every essay is a *reported scene with a named person, present tense*. If a paragraph could sit on Wikipedia, cut it.
3. **The witness is load-bearing — labour, memory, or risk. Never a priest/sadhu "for mood."** No costume, no piety-for-camera.
4. **Photograph the periphery, never the postcard.** Haridwar = the sweeper, not the aarti. Varanasi = the flame, not the pyre. Kumbh = one widow, not the drone. The cliché shot is banned even "tastefully."
5. **Earn grief through restraint (karuna), never NGO shock.** No dead-fish hero, no children-in-filth, no before/after guilt diptych.
6. **Specificity is the anti-cliché** — a real name, date, number (the glacier's retreat in metres, the collapsed catch, the river-km). Generic devotional poetry is worse than cold prose.
7. **Each place earns its slot twice** — by position on the descent (km/temperature) **and** by carrying one FACE wing without becoming a brochure for it.

**The single banned idea:** "lead every place with the river at golden hour and a holy man." That is the Incredible-India template. We are documentarians, not a tourism board. Also banned: travel-guide language, tourism copy, Wikipedia summary, religious preaching, generic spirituality.

## Place-page structure (the template)

`Arrival (hero: encounter + the query answer) → The place told (essay) → THE WITNESS (the one peak) → "This place lives in the book" → Keep a lamp → The record deepens (Archive layer) → Continue the pilgrimage (lateral links) → Walk with the river (email) → Footer.`

**Information order:** answer → encounter → emotion → ask → scholarship → exit. **LIVE photography opens; ARCHIVE deepens.** Search visitors arrive cold — the encounter and the answer share the first screen.

Concentrate effort. **~10–15 flagship, ~30–40 standard, the rest stubs.** Launch all 75 as honest `forthcoming` stubs (the spine is live) → deepen by reach.

## The encounter checklist (the publish gate — `editorialOps.checklist`)

Every flagship/standard page must pass **5/5** before `ready`:
1. one named witness with **consent on file**
2. a hero that is an **encounter** (proximity, available light, the worn/the human), not a vista
3. exactly **one** emotional peak
4. a **distinct truth** no other place repeats
5. **≥1 verifiable** number or date

> *If a draft reads like it explains the river, it goes back. If there's a line the writer is proud of, suspect it. If there's a line the witness could have said, keep it.*

## FACE GANGA framing

Four wings threaded through the archive, not four doors. **Festivals** = preparation/aftermath/periphery, never peak spectacle. **Art** = the living culture (music, manuscript). **Craft** = the maker's hand, process over product. **Environment** = stewardship/continuity register (or, where earned, the **obituary** register) — gravity, never shock.

## Sankalp language

Never "Donate." Use **Sankalp · Keep a Lamp · Support the Journey · Keep a standing lamp** (recurring). Intention-first; amount framed as *what it sustains*; 80G at confirmation only. The lamp is inscriptional; it floats downstream and persists (see the Offering Released moment).

## Production workflow

**State machine (`editorialOps.state`):**
`idea → reporting → drafting → edit → fact-check → translation → ready → published` (+ `needs-update`, `archived`). The **visual track runs in parallel** (`photo-brief → shoot/source → select → grade → consent → final`) and converges at `ready`.

| State | Owner | Gate to advance |
|---|---|---|
| idea | ME | brief + a *distinct truth* approved; assigned |
| reporting | Writer | **witness identified + consent secured** + shoot scheduled |
| drafting | Writer | EN draft, present tense, one peak |
| edit | ME → EiC | structural + voice pass |
| fact-check | Fact-checker | claims log verified + sensitivity read |
| translation | Translator | HI transcreated final, or fallback flagged |
| ready | EiC | encounter checklist 5/5 + consent + hero final |
| published | EiC/ME | gated publish action |

### Witness workflow
Source for labour/memory/risk → **informed consent in the person's own language** (web + book, both languages, permanent, international; right to withdraw) → signed/recorded release filed to `consent.form` → interview (open with their *work*, record + transcribe) → select the one load-bearing quote (EiC sign-off) → `scope: anon` where naming would endanger/shame → **withdrawal = hard publish-block + `needs-update`.** No consent, no publish.

### Photography workflow
Lock the **LUT + shot brief first** (warm-desaturated 25–40% chroma, bone/ink-mapped, film grain, 4:5 & 3:2, eye-level, available light, named subject, act-not-pose). Source in tiers: **public-domain archive** (launches the ARCHIVE layer now) → **commission** ~12–15 flagship LIVE sets + witness portraits (batched by reach) → **strictly-curated documentary licensing** as a bridge (never travel stock). Identifiable people need `subjectConsent`. Every image stores a **provenance & ethics ledger** (source, license, subject, consent, mode, km). **Banned frames:** golden-hour travel poster, drone spectacle, levitating diya, aarti-crowd stock, sadhu-as-vibe, pyres, NGO shock, texture overlays.

### Fact-checking workflow
A **claims log** (`editorialOps.factCheck.claims`): every assertion → a source; quotes checked against the transcript. An independent checker (never the writer) marks `verified`/`unverified`/`corrected`. The **"≥1 verifiable number/date" rule** is a checklist item. Environment/karuna claims get the strictest sourcing. A **cultural sensitivity read** (advisor) is a hard boolean. Public, dated corrections; an error → `needs-update`.

### Translation workflow
Translate the **finished, fact-checked** EN. **Transcreation, not literal translation** — a native Hindi writer-editor recreates register and rhythm in Devanagari (the Hindi reads as authored). **Partial-translation fallback:** publish EN-complete with `hi: 'fallback'`; HI follows. Devanagari typographic care is part of the brief. Chrome strings live in `siteSettings`; content per-document.

### Publication workflow
Gate (enforced by the custom action): `ready` + checklist 5/5 + `i18n.en==='final'` + `factCheck.passed` + consent green + hero final. Stubs publish `forthcoming` → `noindex,follow`. Publish now or **scheduled**. Webhook → `revalidateTag`. Slugs immutable post-publish (renames → 301). Lifecycle: `needs-update` for yearly review (festival dates, environment figures, consent), corrections, or deepening a tier.
